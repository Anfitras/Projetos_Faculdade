Se você não sobrescrever, o Java usará o endereço de memória do objeto como regra. Isso significa que dois objetos com dados idênticos (exemplo: dois clientes com o mesmo CPF) serão tratados como pessoas totalmente diferentes. O Java os jogará em gavetas separadas (hashCode errado) ou não reconhecerá que são a mesma pessoa ao procurar na gaveta (equals errado). O resultado é que se perderá informações dentro da estrutura ou criará dados duplicados sem querer.
